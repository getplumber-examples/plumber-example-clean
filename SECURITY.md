# Security Policy

## Supported versions

The latest minor release and the previous one receive security patches.
Older releases do not.

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a vulnerability

Please report vulnerabilities privately through GitHub's
**Security -> Report a vulnerability** flow, or email security@acme.example.

We acknowledge reports within 48 hours and aim to ship a fix (or a coordinated
disclosure plan) within 14 days. Please do not open a public issue for security
reports.
